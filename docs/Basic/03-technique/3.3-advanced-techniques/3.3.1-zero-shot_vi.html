3.3.1 Zero-shot Prompting: Nghệ thuật hỏi trực tiếp

Sau bài học này, bạn sẽ nắm được

  • Định nghĩa và nguyên lý hoạt động của Zero-shot
  • Các bối cảnh phù hợp để sử dụng Zero-shot
  • Cách viết Prompt Zero-shot chất lượng cao
  • Hạn chế của Zero-shot và phương pháp đối phó

Zero-shot Prompting là gì

Zero-shot (Không mẫu/Không ví dụ) nghĩa là: Không đưa ra bất kỳ ví dụ nào, trực tiếp yêu cầu AI thực hiện nhiệm vụ.

Đây là cách đặt câu hỏi trực tiếp nhất. Bạn mô tả nhiệm vụ, AI dựa vào kiến thức đã học trong quá trình huấn luyện để đưa ra câu trả lời trực tiếp.

Chuyển đổi đoạn code Python 2 sau sang cú pháp Python 3:
print "Hello, World!"

AI không cần bạn diễn giải "chuyển đổi như thế nào", nó đã "biết" sự khác biệt giữa Python 2 và Python 3. Đây chính là cốt lõi của Zero-shot: Tận dụng kiến thức có sẵn của AI để hoàn thành nhiệm vụ.

Tại sao Zero-shot hiệu quả

Các mô hình AI hiện đại đã "nhìn thấy" lượng dữ liệu khổng lồ trong quá trình huấn luyện: code, tài liệu, hướng dẫn, hỏi đáp. Đối với các nhiệm vụ phổ biến, nó đã có khả năng thực thi, chỉ cần bạn mô tả nhiệm vụ rõ ràng.

Giống như nhờ một đồng nghiệp có nhiều kinh nghiệm giúp đỡ —— bạn không cần dạy anh ấy cách viết code, chỉ cần nói rõ bạn muốn gì.

Bối cảnh phù hợp

Bối cảnh Ví dụ Tại sao phù hợp
Chuyển đổi code Python 2 → Python 3 AI quen thuộc với cả hai cú pháp
Phân loại văn bản Phán đoán cảm xúc của bình luận Nhiệm vụ NLP phổ biến
Chuyển đổi định dạng JSON → YAML Định dạng chuẩn, quy tắc rõ ràng
Tạo nội dung Viết một đoạn mô tả sản phẩm Nhiệm vụ sáng tạo, không cần ví dụ
Giải thích code Giải thích đoạn code này làm gì AI có khả năng hiểu code mạnh

Phán đoán bằng một câu: Nếu nhiệm vụ này "AI chắc là biết", thì dùng Zero-shot.

Prompt dở vs Prompt tốt

❌ Prompt dở

Giúp tôi xử lý dữ liệu này

Vấn đề: Nhiệm vụ không rõ ràng. Xử lý nghĩa là gì? Làm sạch? Chuyển đổi? Phân tích?

✅ Prompt tốt

Chuyển đổi dữ liệu CSV sau sang định dạng JSON, mỗi dòng là một đối tượng:

name,age,city
Trương Tam,25,Bắc Kinh
Lý Tứ,30,Thượng Hải

Điểm cải tiến:

  • Nhiệm vụ rõ ràng: CSV → JSON
  • Định dạng rõ ràng: Mỗi dòng một đối tượng
  • Cung cấp dữ liệu đầu vào

So sánh đầu ra

Prompt dở có thể nhận được: "Vui lòng cung cấp dữ liệu cụ thể và nhu cầu xử lý" (AI không thể thực thi)

Prompt tốt nhận được:

[
  { "name": "Trương Tam", "age": 25, "city": "Bắc Kinh" },
  { "name": "Lý Tứ", "age": 30, "city": "Thượng Hải" }
]

4 yếu tố để viết tốt Zero-shot

1. Động từ nhiệm vụ rõ ràng

Mơ hồ Rõ ràng
Xử lý đoạn code này Refactor đoạn code này, tách logic lặp lại thành hàm
Xem dữ liệu này Phân tích giá trị bất thường của dữ liệu này
Làm lại định dạng Chuyển đổi sang định dạng bảng Markdown

2. Yêu cầu đầu ra cụ thể

❌ "Cho tôi một hàm"
✅ "Cho tôi một hàm JavaScript, nhận tham số mảng, trả về mảng đã loại bỏ phần tử trùng lặp"

3. Điều kiện ràng buộc cần thiết

❌ "Viết một thuật toán sắp xếp"
✅ "Viết một thuật toán sắp xếp, yêu cầu: Độ phức tạp thời gian O(n log n), không dùng phương thức sort có sẵn"

4. Dữ liệu đầu vào rõ ràng

Nếu nhiệm vụ cần xử lý dữ liệu, hãy cung cấp trực tiếp trong Prompt.

Kỹ thuật nâng cao: Tăng cường hiệu quả Zero-shot

Thiết lập vai trò (Role Prompting)

Gán cho AI một vai trò, có thể khiến nó trả lời câu hỏi từ một góc nhìn cụ thể.

Bạn là một kỹ sư Frontend có 10 năm kinh nghiệm.
Vui lòng xem xét code React sau đây, chỉ ra các vấn đề hiệu năng tiềm ẩn:
[Code]

Chỉ định định dạng đầu ra

Nói rõ cho AI biết định dạng bạn muốn.

Phân tích phản hồi người dùng sau đây, trích xuất các vấn đề then chốt.
Định dạng đầu ra:
- Vấn đề 1: [Mô tả]
- Vấn đề 2: [Mô tả]
...

Ví dụ thực chiến: Dự án danh sách việc cần làm

Tiểu Lý muốn nhờ AI viết giúp cậu ấy một hàm định dạng ngày tháng.

Prompt Zero-shot

Viết một hàm JavaScript formatDate, chức năng như sau:
- Đầu vào: Đối tượng Date
- Đầu ra: Chuỗi đã định dạng, định dạng là "YYYY năm MM tháng DD ngày"
- Ví dụ: Đầu vào new Date('2024-03-15'), đầu ra "2024 năm 03 tháng 15 ngày"
- Yêu cầu: Tháng và ngày nếu thiếu hai chữ số thì thêm số 0

Đầu ra của AI

function formatDate(date) {
  const year = date.getFullYear();
  const month = String(date.getMonth() + 1).padStart(2, "0");
  const day = String(date.getDate()).padStart(2, "0");
  return `${year} năm ${month} tháng ${day} ngày`;
}

Hạn chế của Zero-shot

Hạn chế Biểu hiện Giải pháp
Định dạng không ổn định Mỗi lần xuất ra định dạng không thống nhất Dùng Few-shot cung cấp ví dụ
Suy luận phức tạp bị lỗi Logic nhiều bước dễ bị nhảy cóc Dùng Chain of Thought
Không hiểu nhu cầu đặc thù Đầu ra không phù hợp quy phạm dự án Bổ sung ngữ cảnh trong Prompt

Tiêu chuẩn phán đoán: Nếu Zero-shot liên tục hai lần cho đầu ra không ưng ý, hãy cân nhắc đổi sang kỹ thuật khác.

Mẫu có thể sao chép

Mẫu chuyển đổi code

Chuyển đổi code [Ngôn ngữ nguồn/Phiên bản] sau đây sang [Ngôn ngữ mục tiêu/Phiên bản]:

[Nội dung code]

Yêu cầu:

- Giữ nguyên chức năng
- Tuân thủ Best Practice của [Ngôn ngữ mục tiêu]

Mẫu tạo nội dung

Bạn là [Định vị vai trò].

Nhiệm vụ: [Mô tả nhiệm vụ cụ thể]

Yêu cầu:

- [Yêu cầu 1]
- [Yêu cầu 2]

Định dạng đầu ra: [Giải thích định dạng]

Mẫu review code

Review code sau đây, chỉ ra vấn đề từ góc độ [Chiều review]:

[Nội dung code]

Định dạng đầu ra:

1. Vấn đề: [Mô tả] | Vị trí: [Số dòng] | Đề xuất: [Phương án sửa đổi]

Trọng điểm bài học

✓ Bản chất của Zero-shot: Tận dụng kiến thức có sẵn của AI, thực hiện trực tiếp nhiệm vụ

✓ Bối cảnh áp dụng: Nhiệm vụ mà AI "chắc là biết" —— chuyển đổi, phân loại, tạo nội dung

✓ Chìa khóa viết tốt Zero-shot: Động từ rõ ràng + Đầu ra cụ thể + Ràng buộc cần thiết

✓ Hạn chế: Định dạng không ổn định, suy luận phức tạp dễ sai

✓ Kỹ thuật tăng cường: Thiết lập vai trò, chỉ định định dạng đầu ra
