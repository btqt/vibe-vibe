# 5.6.1 Ôn tập các điểm cốt lõi của chương

Qua bài học này, bạn sẽ nhận được:

- Sơ đồ cấu trúc kiến thức hoàn chỉnh của chương 5
- Bảng tra cứu các khái niệm cốt lõi (có thể lưu lại)
- Danh sách đối chiếu kỹ năng

## Cấu trúc kiến thức chương 5

```
Chương 5: Từ "Dùng được" đến "Dùng tốt"
│
├── 5.1 Thuốc hối hận (Quản lý phiên bản)
│   ├── Chức năng lịch sử của AI IDE (Không rào cản)
│   ├── Quản lý phiên bản thủ công (Đơn giản hiệu quả)
│   └── GitHub Desktop (Chuẩn bị cho bản nâng cao)
│
├── 5.2 Ra biển lớn (Triển khai trực tuyến)
│   ├── Zeabur Triển khai một cú nhấp (Ưu tiên người dùng Việt Nam/Châu Á)
│   ├── Vercel Triển khai một cú nhấp (Người dùng quốc tế)
│   └── Giải pháp truy cập ổn định
│
├── 5.3 Hào lũy (Ý thức an toàn)
│   ├── Ba lệnh cấm tuyệt đối
│   ├── Cấu hình .env và .gitignore
│   ├── Phòng chống Slopsquatting
│   └── Luôn luôn rà soát code
│
├── 5.4 Tái tiến hóa (Liên tục lặp lại)
│   ├── Phương pháp thu thập phản hồi
│   ├── Quy trình mở rộng tính năng
│   └── Từ dự án đến Portfolio
│
└── 5.5 Biết biên giới (Biên giới năng lực)
    ├── Vibe Coding sở trường gì
    ├── Hạn chế của Vibe Coding
    └── Nhận biết tín hiệu nâng cấp
```

## Bảng tra cứu khái niệm cốt lõi

Bảng này tổng hợp các khái niệm quan trọng nhất của chương 5, có thể lưu lại để dùng dần:

| Khái niệm               | Giải thích một câu                                                      | Tại sao quan trọng                                           |
| ----------------------- | ----------------------------------------------------------------------- | ------------------------------------------------------------ |
| **Quản lý phiên bản**   | Cài "cỗ máy thời gian" cho code, có thể quay lại quá khứ bất cứ lúc nào | Sửa hỏng không lo, mạnh dạn thử nghiệm                       |
| **Commit**              | Một lần "lưu game", ghi lại trạng thái code tại thời điểm đó            | Mỗi lần lưu là một điểm an toàn                              |
| **Triển khai (Deploy)** | Đưa code từ máy lên máy chủ, để người khác truy cập được                | Từ "tự chơi" đến "chia sẻ với thế giới"                      |
| **Tên miền (Domain)**   | "Số nhà" của trang web, ví dụ example.com                               | Để người khác tìm thấy tác phẩm của bạn                      |
| **File .env**           | File cấu hình chứa mật khóa và thông tin nhạy cảm                       | Bảo vệ quyền riêng tư và an toàn tài khoản                   |
| **.gitignore**          | Nói cho Git biết file nào không cần tải lên                             | Ngăn chặn lộ lọt thông tin nhạy cảm                          |
| **Slopsquatting**       | AI gợi ý thư viện không tồn tại, bị kẻ xấu đăng ký trước                | Mối đe dọa an toàn kiểu mới 2024-2025                        |
| **Lặp lại (Iteration)** | Dựa trên phản hồi liên tục cải tiến sản phẩm                            | Sản phẩm tốt là sửa mà thành, không phải làm một lần là xong |
| **README**              | Tài liệu hướng dẫn dự án, giới thiệu là gì, dùng thế nào                | Để người khác (và bản thân trong tương lai) hiểu dự án       |

## Danh sách đối chiếu kỹ năng

Sau khi hoàn thành chương 5, hãy đối chiếu kiểm tra các kỹ năng bạn đã nắm được:

### 5.1 Quản lý phiên bản

| Kỹ năng                       | Tiêu chuẩn nắm vững                                | Tự đánh giá |
| ----------------------------- | -------------------------------------------------- | ----------- |
| Dùng chức năng lịch sử AI IDE | Có thể tìm và khôi phục về phiên bản code trước đó | ☐           |
| Sao lưu dự án thủ công        | Biết copy cả thư mục dự án ở các mốc quan trọng    | ☐           |
| Sử dụng GitHub Desktop        | Có thể hoàn thành thao tác Clone, Commit, Push     | ☐           |
| Viết thông tin Commit rõ ràng | Có thể nói rõ lần này sửa gì trong một câu         | ☐           |

### 5.2 Triển khai trực tuyến

| Kỹ năng                           | Tiêu chuẩn nắm vững                                 | Tự đánh giá |
| --------------------------------- | --------------------------------------------------- | ----------- |
| Chọn nền tảng triển khai phù hợp  | Biết Zeabur, Vercel phù hợp với đối tượng nào       | ☐           |
| Hoàn thành triển khai một cú nhấp | Có thể đưa dự án từ GitHub lên nền tảng             | ☐           |
| Có được liên kết truy cập         | Dự án có địa chỉ web truy cập công khai             | ☐           |
| Xác minh triển khai thành công    | Truy cập bình thường trên cả điện thoại và máy tính | ☐           |

### 5.3 Ý thức an toàn

| Kỹ năng                    | Tiêu chuẩn nắm vững                                                | Tự đánh giá |
| -------------------------- | ------------------------------------------------------------------ | ----------- |
| Nằm lòng ba lệnh cấm       | Không nói cho AI mật khóa, không hardcode khóa, không tải lên .env | ☐           |
| Cấu hình .gitignore        | Biết thêm .env và node_modules vào danh sách bỏ qua                | ☐           |
| Xác minh thư viện AI gợi ý | Lên trang quản lý gói chính thức xác nhận tồn tại trước khi cài    | ☐           |
| Thói quen rà soát code     | Không tin tưởng mù quáng vào code AI tạo ra                        | ☐           |

### 5.4 Liên tục lặp lại

| Kỹ năng                      | Tiêu chuẩn nắm vững                                   | Tự đánh giá |
| ---------------------------- | ----------------------------------------------------- | ----------- |
| Thu thập phản hồi người dùng | Biết để người dùng thật dùng thử và ghi lại vấn đề    | ☐           |
| Quy hoạch mở rộng tính năng  | Có thể dùng khung S.C.A.F.F. để AI thêm tính năng mới | ☐           |
| Sắp xếp Portfolio            | Có thể viết một file README rõ ràng                   | ☐           |

### 5.5 Nhận thức biên giới

| Kỹ năng                     | Tiêu chuẩn nắm vững                                            | Tự đánh giá |
| --------------------------- | -------------------------------------------------------------- | ----------- |
| Biết sở trường là gì        | Rõ ràng Vibe Coding hợp làm công cụ cá nhân, web tĩnh          | ☐           |
| Biết hạn chế ở đâu          | Rõ ràng backend phức tạp, hệ thống người dùng cần học nâng cao | ☐           |
| Nhận biết tín hiệu nâng cấp | Biết khi nào nên học kiến thức sâu hơn                         | ☐           |

## Nhắc nhở các hiểu lầm thường gặp

Dựa trên việc học các chương trước, dưới đây là một số cái hố dễ sa chân:

| Hiểu lầm                                               | Cách làm đúng                                          |
| ------------------------------------------------------ | ------------------------------------------------------ |
| "Đợi code hoàn hảo rồi mới Commit"                     | Bán thành phẩm cũng có thể Commit, sửa bất cứ lúc nào  |
| "Triển khai phiền phức, cứ dùng local đã"              | Triển khai chỉ mất vài phút, làm ngay bây giờ          |
| "Code AI tạo ra chắc không sao đâu"                    | Luôn luôn rà soát, đặc biệt là phần liên quan an toàn  |
| "Bản cơ bản đơn giản quá, tôi muốn học thẳng nâng cao" | Phương pháp luận của bản cơ bản là gốc rễ của nâng cao |
| "Thứ tôi làm đơn giản quá, ngại chia sẻ"               | Giải quyết được vấn đề là đồ tốt, mạnh dạn chia sẻ     |

## Điểm cốt lõi của bài này

- Cốt lõi của chương 5 là khiến tác phẩm của bạn "từ dùng được đến dùng tốt"
- Quản lý phiên bản, Triển khai, An toàn, Lặp lại, Nhận thức biên giới — 5 năng lực không thể thiếu
- Đối chiếu danh sách kỹ năng để kiểm tra mức độ nắm vững của bản thân
- Tránh các hiểu lầm thường gặp, xây dựng thói quen đúng đắn
