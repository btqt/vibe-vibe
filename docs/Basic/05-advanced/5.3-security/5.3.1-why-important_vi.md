# 5.3.1 Tại sao phần này rất quan trọng

## Trường hợp thực tế: Mất hàng nghìn đô la trong 10 phút

Đây là kịch bản lặp đi lặp lại trong năm 2024-2025:

1. Lập trình viên dùng AI tạo code
2. AI viết trực tiếp khóa API (API Key) vào trong code
3. Code được tải lên GitHub (ngay cả kho chứa riêng tư cũng không an toàn)
4. **Vài phút sau, tài khoản bị trộm sạch**

Tháng 11 năm 2024, dự án blockchain Pump Science chỉ vì rò rỉ khóa trên GitHub, dẫn đến kẻ tấn công tạo ra các token lừa đảo, gây thiệt hại cho người dùng.

## Sự kiện trộm cắp chứng thực quy mô lớn

Tháng 10 năm 2024, các nhà nghiên cứu an ninh phát hiện chiến dịch tấn công mang mã hiệu **EMERALDWHALE**:

- Kẻ tấn công đã quét khoảng **500 triệu địa chỉ IP**
- Trộm hơn **15,000 chứng thực dịch vụ đám mây** từ các file cấu hình Git bị lộ
- Liên quan đến **10,000+ kho chứa code riêng tư**
- Chứng thực bị trộm được dùng cho tấn công lừa đảo (phishing) và thư rác

Những chứng thực này bao gồm khóa AWS, chuỗi kết nối cơ sở dữ liệu, API Token - đều là những thứ lập trình viên "vô tình" commit vào kho code.

## Tại sao thời đại lập trình AI rủi ro cao hơn

Còn nhớ "ảo giác AI" đã nhắc đến ở chương 3 không? Trong lĩnh vực an ninh, sự "không đáng tin cậy" của AI có thể trực tiếp dẫn đến:

| Hành vi của AI                       | Hậu quả an toàn        |
| ------------------------------------ | ---------------------- |
| Viết khóa vào code dưới dạng "ví dụ" | Rò rỉ khóa             |
| Đề xuất gói phần mềm không tồn tại   | Có thể cài phải mã độc |
| Bỏ qua xác thực đầu vào              | Để lại lỗ hổng an ninh |
| Sử dụng phương thức mã hóa lỗi thời  | Dữ liệu dễ bị bẻ khóa  |

Theo báo cáo Veracode 2025, khi AI lựa chọn giữa "cách viết an toàn" và "cách viết không an toàn", **45% trường hợp nó chọn cách không an toàn**.

## Tin tốt

**Chỉ cần biết luật chơi, bạn có thể tránh được dễ dàng.**

Các bài tiếp theo sẽ dạy bạn:

- Những điều tuyệt đối không được làm (Ba lệnh cấm lớn)
- Cách kiểm tra các mối nguy an toàn trong code
- Cách nhận biết các gói phần mềm đáng ngờ do AI đề xuất
- Cách hình thành thói quan rà soát an toàn

Những kiến thức này sẽ giúp bạn an toàn hơn 90% người mới bắt đầu.
