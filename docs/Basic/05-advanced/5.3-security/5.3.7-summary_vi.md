# 5.3.7 Tóm tắt các điểm chính

Chúc mừng bạn đã hoàn thành bài học về an toàn. Giờ đây bạn đã nắm vững những kiến thức an toàn quan trọng nhất trong thời đại lập trình AI.

## Ba lệnh cấm lớn

Ghi nhớ 3 ranh giới đỏ này, có thể tránh được 90% các vấn đề an toàn:

| Lệnh cấm                          | Giải thích                                          |
| --------------------------------- | --------------------------------------------------- |
| ❌ Không nói cho AI mật khẩu/khóa | Dù là "giúp tôi kiểm tra định dạng" cũng không được |
| ❌ Không viết khóa vào trong code | Sử dụng file .env và biến môi trường                |
| ❌ Không tải lên file .env        | Đảm bảo cấu hình .gitignore chính xác               |

## Hai thói quen tốt

1. **Xác minh trước khi cài đặt thư viện mới**: Lên npmjs.com hoặc pypi.org xác nhận gói tồn tại và đáng tin cậy
2. **Kiểm tra nhanh trước khi commit**: Tìm kiếm các từ khóa `password`, `secret`, `key` v.v.

## Một nguyên tắc cốt lõi

::: tip Nguyên tắc cốt lõi
**Code do AI tạo ra, trách nhiệm cuối cùng thuộc về bạn.**

Duy trì thói quen rà soát, không tin tưởng mù quáng vào đầu ra của AI.
:::

## Bản đồ kiến thức phần này

```
5.3 Ý thức an toàn
├── 5.3.0 Những con số giật mình (Xây dựng cảm giác cấp bách)
├── 5.3.1 Tại sao rất quan trọng (Trường hợp thực tế)
├── 5.3.2 Vùng cấm (Những thứ không nói với AI)
├── 5.3.3 Mối nguy trong code (.env và .gitignore)
├── 5.3.4 Danh sách kiểm tra an toàn (Các mục kiểm tra khả thi)
├── 5.3.5 Slopsquatting (Xác minh thư viện AI đề xuất)
└── 5.3.6 Luôn rà soát code (Xây dựng thói quen rà soát)
```

## Dự báo bản nâng cao

Phần này chỉ là nhập môn an toàn. Trong bản nâng cao, bạn sẽ học thêm nhiều kiến thức an toàn cao cấp hơn:

| Chủ đề                 | Bạn sẽ học                                  |
| ---------------------- | ------------------------------------------- |
| Chứng thực và Ủy quyền | JWT, OAuth 2.0, quản lý phiên (session)     |
| An toàn API            | CORS, XSS, Phòng chống CSRF                 |
| An toàn dữ liệu        | Mã hóa đường truyền, xử lý dữ liệu nhạy cảm |
| Kiểm toán an toàn      | Ghi nhật ký (log), phát hiện bất thường     |

## Lời cuối

> Kiến thức an toàn không phải dùng để dọa người, mà là dùng để bảo vệ bạn.
>
> Chỉ cần hình thành thói quan tốt, bạn có thể yên tâm sử dụng AI lập trình để tạo ra giá trị.
