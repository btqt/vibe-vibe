5.2.8 Các câu hỏi thường gặp về triển khai

Việc gặp vấn đề trong quá trình triển khai là bình thường. Dưới đây là tổng hợp các vấn đề thường gặp và giải pháp.

Bảng tra cứu nhanh

Vấn đề Nguyên nhân có thể Cách giải quyết
Triển khai thất bại Code có lỗi cú pháp Xác nhận chạy bình thường ở máy cục bộ trước
Trang trắng xóa Đường dẫn file đầu vào sai Đảm bảo index.html nằm ở thư mục gốc dự án
Mất giao diện (style) Đường dẫn CSS dùng đường dẫn tuyệt đối Đổi sang đường dẫn tương đối (như ./style.css)
Không hiện ảnh Đường dẫn ảnh sai hoặc sai chữ hoa chữ thường Kiểm tra đường dẫn và chữ hoa chữ thường của tên file
Chức năng không chạy Lỗi JavaScript Mở bảng điều khiển trình duyệt (F12) xem lỗi
Đại lục không truy cập được Nhiễm độc DNS Đổi sang dùng Zeabur hoặc Maozi Cloud
Tải rất chậm File tài nguyên quá lớn Nén ảnh, kiểm tra xem có file lớn không cần thiết không
Cập nhật xong không thay đổi Bộ nhớ đệm trình duyệt Làm mới bắt buộc (Ctrl+F5) hoặc xóa bộ nhớ đệm

Giải thích chi tiết vấn đề

Vấn đề 1: Triển khai thất bại, báo lỗi build

Hiện tượng: Sau khi nhấp triển khai, trạng thái cứ hiển thị "Building" rồi chuyển thành "Failed".

Các bước rà soát:

  1. Xem nhật ký build, tìm thông tin lỗi màu đỏ
  2. Kiểm tra dự án ở máy cục bộ có chạy bình thường không
  3. Xác nhận tất cả các file cần thiết đã được đẩy lên GitHub

Nguyên nhân thường gặp:

  • Code có lỗi cú pháp
  • Thiếu file cấu hình cần thiết
  • Phụ thuộc chưa được cài đặt đúng cách

Mẫu cầu cứu AI:

Tôi đang triển khai dự án trên [Zeabur/Vercel], thì bị lỗi build.

Nhật ký build hiển thị lỗi này:
[Dán thông tin lỗi màu đỏ]

Dự án của tôi là [Web tĩnh thuần HTML/CSS/JS / Dự án React / ...].
Chạy cục bộ [bình thường / cũng có vấn đề].

Hãy giúp tôi phân tích nguyên nhân và đưa ra giải pháp.

Vấn đề 2: Trang trắng xóa

Hiện tượng: Triển khai thành công, nhưng mở địa chỉ web ra là trang trắng.

Các bước rà soát:

  1. Mở công cụ nhà phát triển trình duyệt (F12)
  2. Xem Console (Bảng điều khiển) có lỗi đỏ không
  3. Xem Network (Mạng) có lỗi 404 không

Nguyên nhân thường gặp:

  • index.html không nằm ở thư mục gốc
  • Đường dẫn file JavaScript sai dẫn đến script không tải được
  • Sử dụng đường dẫn tuyệt đối, môi trường online không tìm thấy file

Cách giải quyết:

<!-- Sai: Đường dẫn tuyệt đối -->
<script src="/js/app.js"></script>

<!-- Đúng: Đường dẫn tương dối -->
<script src="./js/app.js"></script>

Vấn đề 3: Mất giao diện (style)

Hiện tượng: Trang hiển thị được, nhưng giao diện vỡ hoàn toàn, hoặc không có style.

Nguyên nhân thường gặp:

  • Đường dẫn file CSS sai
  • File CSS chưa được đẩy lên GitHub
  • Sử dụng đường dẫn tuyệt đối

Cách giải quyết:

<!-- Sai -->
<link rel="stylesheet" href="/style.css" />

<!-- Đúng -->
<link rel="stylesheet" href="./style.css" />

Vấn đề 4: Chức năng không hoạt động

Hiện tượng: Trang hiển thị bình thường, nhưng nhấp nút không phản ứng, chức năng không chạy.

Các bước rà soát:

  1. Mở bảng điều khiển trình duyệt (F12 → Console)
  2. Xem có thông tin lỗi màu đỏ không
  3. Thử kích hoạt chức năng, xem có lỗi mới xuất hiện không

Mẫu cầu cứu AI:

Dự án danh sách việc cần làm của tôi sau khi triển khai, chức năng [thêm nhiệm vụ / xóa nhiệm vụ / ...] không hoạt động.

Bảng điều khiển trình duyệt hiển thị lỗi này:
[Dán thông tin lỗi]

Khi chạy cục bộ chức năng vẫn bình thường.
Hãy giúp tôi phân tích nguyên nhân.

Vấn đề 5: Sau khi cập nhật trang web không thay đổi

Hiện tượng: Code đã đẩy lên GitHub, nhưng trang web vẫn là phiên bản cũ.

Nguyên nhân có thể:

  1. Triển khai chưa hoàn tất (vào bảng điều khiển xem trạng thái)
  2. Bộ nhớ đệm trình duyệt

Cách giải quyết:

  • Đợi triển khai hoàn tất (thường 1-2 phút)
  • Làm mới bắt buộc trang: Windows nhấn Ctrl+F5, Mac nhấn Cmd+Shift+R
  • Mở chế độ ẩn danh/riêng tư để truy cập
