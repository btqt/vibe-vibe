# 5.2.5 Các thao tác thiết thực sau khi triển khai

Triển khai thành công chỉ là bắt đầu. Bạn sẽ liên tục cải tiến dự án của mình, cần biết cách cập nhật phiên bản trực tuyến, cách xem trạng thái triển khai, cũng như cách quay lại (rollback) khi gặp sự cố.

## Cách cập nhật trang web

Đây là thao tác thường dùng nhất. Khi bạn sửa code ở máy cục bộ và muốn phiên bản trực tuyến cũng được cập nhật, bạn cần làm:

```
Sửa code cục bộ → git add → git commit → git push → Tự động triển khai
```

Các bước cụ thể:

1. Sửa code của bạn ở máy cục bộ (ví dụ đổi màu một nút bấm)
2. Sử dụng GitHub Desktop gửi thay đổi (Commit)
3. Nhấp "Push origin" đẩy lên GitHub
4. Đợi 1-2 phút
5. Tải lại trang web của bạn, xem thay đổi

**Bạn không cần** đăng nhập lại nền tảng triển khai, cũng không cần nhấp bất kỳ nút triển khai nào. Zeabur và Vercel đều sẽ tự động phát hiện cập nhật code của bạn và tự động triển khai lại.

Đây chính là ma thuật của "Triển khai liên tục" (Continuous Deployment).

## Cách xem lịch sử triển khai

Mỗi lần triển khai sau khi cập nhật code đều để lại bản ghi. Bạn có thể xem trong bảng điều khiển của nền tảng triển khai:

### Trong Zeabur

1. Đăng nhập bảng điều khiển Zeabur
2. Nhấp vào dự án của bạn
3. Nhấp vào dịch vụ của bạn
4. Tìm thẻ "Lịch sử triển khai" hoặc "Deployments"

Bạn có thể thấy:

- Thời gian mỗi lần triển khai
- Triển khai thành công hay thất bại
- Git Commit tương ứng với mỗi lần triển khai

### Trong Vercel

1. Đăng nhập Vercel Dashboard
2. Nhấp vào dự án của bạn
3. Nhấp thẻ "Deployments"

Mỗi lần Git Push đều tạo ra một bản ghi triển khai, thậm chí mỗi nhánh (branch) đều có liên kết xem trước độc lập.

## Cách quay lại (rollback) phiên bản trước

Nếu sau khi cập nhật bạn phát hiện trang web bị lỗi, có thể nhanh chóng quay lại phiên bản bình thường trước đó.

### Trong Zeabur

1. Tìm lịch sử triển khai
2. Tìm bản ghi triển khai bình thường trước đó
3. Nhấp "Triển khai lại" hoặc "Redeploy"

### Trong Vercel

1. Tại trang Deployments tìm lần triển khai trước đó
2. Nhấp menu ba chấm bên phải
3. Chọn "Promote to Production"

Việc quay lại là tức thì, thường chỉ vài giây là có hiệu lực.

## Cách xem nhật ký triển khai

Khi triển khai thất bại, nhật ký (log) là chìa khóa để tìm vấn đề.

Trong bản ghi triển khai, nhấp vào một lần triển khai cụ thể, có thể thấy:

- Nhật ký build: Dự án được xây dựng như thế nào
- Nhật ký chạy (Runtime logs): Đầu ra khi dự án đang chạy

Nếu thấy thông tin lỗi màu đỏ, có thể sao chép cho AI giúp bạn phân tích.

## Tên miền tùy chỉnh (Dự báo nâng cao)

Hiện tại bạn đang sử dụng tên miền miễn phí do nền tảng cung cấp:

- Zeabur: `your-project.zeabur.app`
- Vercel: `your-project.vercel.app`

Nếu bạn muốn có tên miền riêng (như `my-todo.com`), cần:

1. Mua tên miền (Aliyun, Tencent Cloud, Cloudflare, v.v.)
2. Gắn tên miền trong nền tảng triển khai
3. Cấu hình bản ghi DNS

Các thao tác này phức tạp hơn một chút, thuộc nội dung bản nâng cao. Đối với việc học và sử dụng cá nhân, tên miền miễn phí là hoàn toàn đủ dùng.

::: tip Dự báo nâng cao
Trong bản nâng cao, bạn sẽ học cách mua và cấu hình tên miền tùy chỉnh, cũng như cách sử dụng Cloudflare để tăng tốc trang web của bạn.
:::
