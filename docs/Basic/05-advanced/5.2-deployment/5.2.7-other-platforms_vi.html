5.2.7 Các lựa chọn triển khai khác

Ngoài Zeabur và Vercel, còn có một số nền tảng triển khai khác để lựa chọn. Dưới đây là giới thiệu sơ lược để tham khảo.

Netlify: Kéo thả để triển khai

Netlify là một nền tảng lưu trữ web tĩnh phổ biến khác, đặc điểm lớn nhất của nó là hỗ trợ "kéo thả để triển khai".

Cách sử dụng

  1. Mở Netlify Drop
  2. Kéo trực tiếp thư mục dự án của bạn vào trang trình duyệt
  3. Đợi vài giây, quá trình triển khai hoàn tất
  4. Nhận được một địa chỉ web *.netlify.app

[Gợi ý hình ảnh] Nội dung: Trang Netlify Drop, hiển thị khu vực kéo thả Prompt: web interface showing drag and drop zone, file folder being dragged, clean modern design, light background, 8k --ar 16:9

Ưu nhược điểm

Ưu điểm Nhược điểm
Thao tác cực đơn giản, chỉ cần kéo thả Cập nhật cần kéo thả lại
Không cần tài khoản GitHub Truy cập tại Đại lục có vấn đề
Triển khai cấp giây Không phù hợp với dự án cập nhật thường xuyên

Phù hợp: Trình diễn tạm thời, dự án dùng một lần, hoặc khi không muốn dùng Git.

Cách triển khai liên tục

Nếu muốn tự động triển khai sau khi cập nhật code, bạn có thể:

  1. Đăng ký tài khoản Netlify
  2. Kết nối kho chứa GitHub
  3. Cấu hình cài đặt build

Như vậy sẽ giống như Zeabur, Vercel, tự động triển khai sau khi Push.

GitHub Pages: Hoàn toàn miễn phí

GitHub Pages là dịch vụ lưu trữ web tĩnh do GitHub cung cấp, hoàn toàn miễn phí, không cần tài khoản bổ sung nào.

Nguyên lý cơ bản

GitHub Pages sẽ xuất bản trực tiếp các file HTML trong kho chứa của bạn thành trang web.

Sơ lược cách sử dụng

  1. Đảm bảo kho chứa của bạn là public (công khai)
  2. Đảm bảo có một file index.html
  3. Vào Settings → Pages của kho chứa
  4. Chọn nhánh muốn triển khai (thường là main)
  5. Đợi vài phút, nhận được địa chỉ web username.github.io/repo-name

Ưu nhược điểm

Ưu điểm Nhược điểm
Hoàn toàn miễn phí Chỉ hỗ trợ web tĩnh
Tích hợp sâu với GitHub Truy cập tại Đại lục không ổn định
Hỗ trợ tên miền tùy chỉnh Cấu hình hơi phức tạp

Phù hợp: Blog cá nhân, trang tài liệu, trang chủ dự án mã nguồn mở.

::: tip Dự báo nâng cao Cấu hình chi tiết của GitHub Pages và tự động triển khai với GitHub Actions sẽ được giảng giải trong bản nâng cao. :::

Railway: Hạn mức miễn phí ít hơn

Railway là nền tảng triển khai ứng dụng fullstack, hỗ trợ triển khai frontend, backend, cơ sở dữ liệu "tất cả trong một".

Đặc điểm:

  • Hỗ trợ nhiều ngôn ngữ và framework
  • Cung cấp dịch vụ cơ sở dữ liệu
  • Hạn mức miễn phí $5 mỗi tháng

Trang chủ: railway.app

Phù hợp: Dự án cần dịch vụ backend. Đối với danh sách việc cần làm thuần frontend của chúng ta, dùng Railway hơi "dùng dao mổ trâu giết gà".

Render: Triển khai fullstack

Render tương tự như Railway, là một nền tảng triển khai fullstack.

Đặc điểm:

  • Tầng miễn phí hỗ trợ web tĩnh
  • Hỗ trợ dịch vụ backend và cơ sở dữ liệu
  • Tự động triển khai từ Git

Trang chủ: render.com

Cloudflare Pages: Triển khai biên (Edge)

Cloudflare Pages là dịch vụ lưu trữ web tĩnh do Cloudflare cung cấp.

Đặc điểm:

  • Hoàn toàn miễn phí
  • Tăng tốc mạng biên toàn cầu
  • Tích hợp sâu với hệ sinh thái Cloudflare

Hạn chế:

  • Truy cập tại Đại lục cần cấu hình thêm
  • Cấu hình phức tạp hơn Zeabur một chút

Trang chủ: pages.cloudflare.com

Cách chọn

Đối với việc học bản cơ bản, không cần đắn đo giữa các lựa chọn này. Khuyên dùng:

  • Người dùng Đại lục: Dùng Zeabur
  • Người dùng quốc tế: Dùng Vercel
  • Trình diễn tạm thời: Dùng Netlify Drop

Đợi bạn học xong bản cơ bản, có nhiều nhu cầu hơn, lúc đó hãy khám phá các nền tảng khác.
