# 5.1.5 Thực hành GitHub Desktop: Thành thạo trong 10 phút

Phần này sẽ dẫn bạn đi từng bước hoàn thành việc cài đặt và thao tác cơ bản với GitHub Desktop.

## Bước 1: Tải về và cài đặt

Truy cập trang chủ GitHub Desktop để tải bộ cài đặt:

**Địa chỉ trang chủ**: [desktop.github.com](https://desktop.github.com)

- Tải phiên bản tương ứng với hệ điều hành của bạn (Windows/Mac)
- Sau khi tải xong, nhấp đúp vào bộ cài đặt, làm theo hướng dẫn để hoàn tất cài đặt

**Bạn sẽ thấy**: Sau khi cài đặt hoàn tất, biểu tượng GitHub Desktop xuất hiện trong danh sách ứng dụng của bạn.

::: warning Lời khuyên cho người dùng trong nước
Truy cập GitHub trong nước có thể khá chậm. Nếu tốc độ tải xuống rất chậm, bạn có thể thử:

1. Đổi thời gian tải xuống (buổi sáng thường nhanh hơn)
2. Sử dụng điểm phát sóng di động (hotspot) điện thoại
3. Tìm kiếm "GitHub Desktop mirror" (link tải dự phòng)

Nếu thực sự không thể tải xuống, bạn có thể tạm thời bỏ qua phần này, dùng phương pháp ở 5.1.2 và 5.1.3 trước.
:::

## Bước 2: Đăng ký/Đăng nhập tài khoản GitHub

Sau khi mở GitHub Desktop, nó sẽ nhắc bạn đăng nhập.

**Nếu bạn chưa có tài khoản GitHub**:

1. Nhấp vào "Create your free account" hoặc truy cập [github.com](https://github.com)
2. Điền tên người dùng, email, mật khẩu
3. Hoàn tất xác minh email

**Nếu bạn đã có tài khoản**:

1. Nhấp vào "Sign in to GitHub.com"
2. Đăng nhập trong trang trình duyệt bật ra
3. Cấp quyền cho GitHub Desktop truy cập tài khoản của bạn

**Bạn sẽ thấy**: Sau khi đăng nhập thành công, giao diện hiển thị "Let's get started!" hoặc trang chào mừng tương tự.

## Bước 3: Biến danh sách việc cần làm thành kho chứa (repository)

Bây giờ chúng ta sẽ biến dự án danh sách việc cần làm đã làm ở chương 4 thành một kho chứa Git.

### Cách 1: Tạo từ thư mục hiện có

1. Nhấp vào **"Create a New Repository on your Hard Drive"** hoặc **"Add an Existing Repository"**

2. Nếu chọn "Create New":
   - **Name**: Nhập tên kho chứa, ví dụ `my-todo-list`
   - **Local Path**: Chọn **thư mục cha** của thư mục dự án của bạn
   - Tích chọn **"Initialize this repository with a README"**
   - Nhấp vào **"Create Repository"**

3. Sao chép các file trong danh sách việc cần làm của bạn vào thư mục kho chứa mới vừa tạo

### Cách 2: Thêm trực tiếp dự án hiện có (Khuyên dùng)

1. Nhấp vào **File** → **Add Local Repository**
2. Chọn thư mục dự án danh sách việc cần làm của bạn
3. Nếu có thông báo "Đây không phải là kho chứa Git", nhấp vào **"create a repository"**
4. Xác nhận tạo

**Bạn sẽ thấy**: Bên trái hiển thị tên dự án của bạn, khu vực giữa hiển thị các thay đổi của file.

## Bước 4: Commit lần đầu tiên (Lưu trữ)

Bây giờ dự án của bạn đã là một kho chứa Git. Hãy tạo điểm lưu trữ đầu tiên.

1. Tại giao diện chính GitHub Desktop, bên trái sẽ hiển thị tất cả các file có thay đổi
2. Nếu là kho chứa mới, tất cả các file sẽ hiển thị là "Mới" (New)

3. Tìm ô nhập **Summary** ở góc dưới bên trái
   - Nhập một câu mô tả, ví dụ: "Bản hoàn chỉnh danh sách việc cần làm chương 4"
   - Description có thể để trống

4. Nhấp vào nút màu xanh **"Commit to main"**

**Bạn sẽ thấy**:

- Danh sách file bên trái đã trống (vì mọi thay đổi đã được lưu)
- Giao diện báo "No local changes"

Chúc mừng! Bạn đã tạo được điểm lưu trữ đầu tiên.

## Bước 5: Xem lịch sử (History)

Hãy xem lại bản lưu trữ vừa tạo.

1. Nhấp vào thẻ **"History"** ở trên cùng (hoặc tìm ở bên trái)
2. Bạn sẽ thấy một dòng thời gian, mỗi nút là một Commit

**Bạn sẽ thấy**:

- Commit vừa rồi xuất hiện trong danh sách
- Hiển thị mô tả bạn đã viết (như "Bản hoàn chỉnh danh sách việc cần làm chương 4")
- Bên phải hiển thị những file nào có trong lần Commit này

## Bước 6: Giả lập làm hỏng rồi quay lại

Hãy thử nghiệm chức năng "đọc bản lưu".

### 6.1 Thực hiện một thay đổi trước

1. Dùng AI IDE hoặc trình soạn thảo văn bản của bạn mở file HTML của danh sách việc cần làm
2. Sửa đại một cái gì đó (ví dụ sửa tiêu đề thành "Thử nghiệm quay lại")
3. Lưu file

### 6.2 Tạo một bản lưu trữ mới

1. Quay lại GitHub Desktop
2. Bạn sẽ thấy thay đổi vừa rồi hiển thị trong danh sách Changes
3. Viết một mô tả: "Thử nghiệm thay đổi"
4. Nhấp vào **"Commit to main"**

### 6.3 Quay lại phiên bản trước đó

1. Nhấp vào thẻ **"History"**
2. Tìm Commit "Bản hoàn chỉnh danh sách việc cần làm chương 4"
3. Nhấp chuột phải vào nó
4. Chọn **"Revert Changes in Commit"** (hoặc **"Revert this Commit"**)
5. Xác nhận thao tác

**Bạn sẽ thấy**:

- Đã tạo một Commit mới, mô tả tương tự "Revert Thử nghiệm thay đổi"
- Mở file HTML, tiêu đề đã khôi phục về nguyên trạng

## Bước 7: Đẩy lên đám mây (Push)

Bước cuối cùng, tải code của bạn lên trang web GitHub.

1. Nhấp vào nút **"Publish repository"** hoặc **"Push origin"** ở trên cùng
2. Trong hộp thoại bật ra:
   - **Name**: Xác nhận tên kho chứa
   - **Description**: Có thể viết một câu mô tả (tùy chọn)
   - **Keep this code private**: Khuyên dùng tích chọn (giữ riêng tư)
3. Nhấp vào **"Publish Repository"**

**Bạn sẽ thấy**:

- Nút bấm đổi thành "Fetch origin"
- Không có thông báo lỗi

### Xác nhận thành công

1. Mở trình duyệt, truy cập [github.com](https://github.com)
2. Đăng nhập tài khoản của bạn
3. Tại trang cá nhân, bạn sẽ thấy kho chứa vừa tạo
4. Nhấp vào xem, có thể thấy các file code của bạn

Chúc mừng! Code của bạn giờ đã có sao lưu trên đám mây. Ngay cả khi máy tính hỏng, cũng có thể tìm lại từ GitHub.

## Các vấn đề thường gặp

| Vấn đề                         | Nguyên nhân có thể                  | Cách giải quyết                                 |
| ------------------------------ | ----------------------------------- | ----------------------------------------------- |
| Push thất bại                  | Vấn đề mạng                         | Kiểm tra kết nối mạng, thử lại sau              |
| Không tìm thấy thư mục dự án   | Vấn đề đường dẫn                    | Xác nhận vị trí thư mục, thêm lại               |
| Đăng nhập thất bại             | Vấn đề tài khoản                    | Thử đăng nhập github.com trên trình duyệt trước |
| Thao tác rất chậm              | Vấn đề mạng (thường gặp trong nước) | Kiên nhẫn chờ đợi, hoặc đổi thời gian khác      |
| Báo lỗi "not a git repository" | Chưa khởi tạo                       | Làm theo Bước 3 để tạo kho chứa                 |

## Danh sách kiểm tra thành công

Hoàn thành tất cả các bước sau, chứng tỏ bạn đã nắm vững thao tác cơ bản của GitHub Desktop:

- [ ] Cài đặt thành công GitHub Desktop
- [ ] Đã đăng nhập tài khoản GitHub
- [ ] Đã biến dự án danh sách việc cần làm thành kho chứa
- [ ] Đã tạo Commit đầu tiên
- [ ] Đã thấy ghi chép Commit trong History
- [ ] Đã thử nghiệm thành công chức năng quay lại
- [ ] Đã đẩy code lên đám mây GitHub
- [ ] Đã thấy dự án của mình trên github.com

Nếu hoàn thành tất cả, chúc mừng bạn — bạn đã học được công cụ quản lý phiên bản mà các lập trình viên chuyên nghiệp sử dụng hàng ngày!
